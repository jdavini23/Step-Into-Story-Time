# Step Into Storytime - Product Roadmap

Tracked improvement ideas identified from a PRD audit on February 8, 2026.

---

## 1. First-Time User Flow (Try Before Sign-Up)
**Priority:** High
**Status:** In Progress

The PRD says a first-time user should be able to create a story before signing up, then get prompted to sign up to save or export it. Currently, the story wizard requires authentication. This blocks users from experiencing the product's value before committing.

**Goal:** Let new visitors create one story without an account, then prompt them to sign up to save it.

---

## 2. Mobile-First Reading Experience
**Priority:** Medium
**Status:** Not Started

The PRD emphasizes a "mobile-first reading experience" as a core UX principle. The story reader and wizard should be audited for mobile usability — layout, font sizes, touch targets, and reading comfort on phones.

**Goal:** Ensure the story reader and wizard feel natural and comfortable on mobile devices.

---

## 3. Story Library Pagination & Filtering
**Priority:** Medium
**Status:** Not Started

The PRD calls for pagination and filtering in the story library. Currently the dashboard loads all stories at once with only a favorites/all toggle. No date filtering, no search, no pagination for large libraries.

**Goal:** Add pagination, search, and date-based filtering so the library scales well for frequent users.

---

## 4. Story Management UI (Delete, Edit)
**Priority:** Low
**Status:** Not Started

The backend has delete and update endpoints for stories. Verify the UI exposes clear management actions (delete, edit titles, etc.) for stories in the library.

**Goal:** Confirm and polish story management actions in the dashboard.

---

## 5. Calm, Bedtime-Friendly Design Audit
**Priority:** Medium
**Status:** Not Started

The PRD emphasizes calm, low-stimulation design. The current UI uses gradients, emojis, and animations that may feel more "marketing energetic" than "bedtime calm." Worth reviewing the overall feel, especially on the dashboard and story reader.

**Goal:** Audit and adjust visual tone to be calmer and more bedtime-appropriate.

---

## 6. Night Mode UI
**Priority:** Low
**Status:** Not Started

Listed as a future consideration in the PRD but aligns closely with the "calm, bedtime-friendly" UX principle. A dark/night mode would reduce screen brightness during actual bedtime use.

**Goal:** Add an optional night/dark mode toggle.
